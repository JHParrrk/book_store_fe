# 📚 프로젝트 라이브러리 총정리 가이드 (패키지 의존성)

이 문서는 `package.json`에 명시된 라이브러리(패키지)들이 이 프로젝트에서 어떤 역할을 하는지, 그리고 얼마나 중요한지를 상세히 설명합니다. (중요도를 잘 모르실 경우를 대비해 별점(⭐️)으로 구분했습니다.)

---

## 1. ⚛️ 필수 코어 및 프레임워크 (Core)

| 패키지명                           |   중요도   | 설명                                                                                                                                               |
| ---------------------------------- | :--------: | -------------------------------------------------------------------------------------------------------------------------------------------------- |
| **react**                          | ⭐️⭐️⭐️⭐️⭐️ | UI를 만들기 위한 핵심 자바스크립트 라이브러리입니다. 프로젝트의 근간입니다.                                                                        |
| **react-dom**                      | ⭐️⭐️⭐️⭐️⭐️ | React로 만든 UI를 실제 웹 브라우저(DOM) 화면에 그려주는(Render) 역할을 합니다.                                                                     |
| **vite**, **@vitejs/plugin-react** | ⭐️⭐️⭐️⭐️⭐️ | (빌드 도구) 예전에는 Webpack을 썼지만, 훨씬 더 빠르고 가벼운 현세대 프로젝트 빌드 도구입니다. 코드를 수정하면 0.1초 만에 브라우저에 반영해 줍니다. |
| **typescript**                     | ⭐️⭐️⭐️⭐️⭐️ | 자바스크립트에 '타입(Type)'을 강제하는 언어입니다. 개발 중 발생할 수 있는 오타/버그를 미리 막아주어 엔터프라이즈 앱에서 필수입니다.                |

---

## 2. 🗂️ 상태 관리 (State Management)

| 패키지명                  |   중요도   | 설명                                                                                                                                                                                           |
| ------------------------- | :--------: | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **@tanstack/react-query** | ⭐️⭐️⭐️⭐️⭐️ | **서버 상태(Server State) 관리 끝판왕**. API 서버에서 데이터를 가져오고, 로딩/에러 처리, 캐싱(저장), 백그라운드 업데이트를 전부 자동화해 줍니다. 없으면 예외 처리를 전부 수동으로 짜야 합니다. |
| **zustand**               |  ⭐️⭐️⭐️⭐️  | **클라이언트(브라우저) 전역 상태 관리**. (예: 현재 로그인한 사용자 정보, 모달창 떴는지 여부 확인). Redux보다 코드가 훨씬 짧고 간단하며 최근 가장 유행하는 상태 관리 툴입니다.                  |

---

## 3. 🚦 화면 이동 및 통신 (Routing & Network)

| 패키지명             |   중요도   | 설명                                                                                                                                                                   |
| -------------------- | :--------: | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **react-router-dom** | ⭐️⭐️⭐️⭐️⭐️ | 브라우저 URL이 (ex: `/login`, `/basket`) 바뀔 때 새로고침 없이 부드럽게 화면의 컴포넌트만 교체해 주는 라우팅 라이브러리입니다.                                         |
| **axios**            | ⭐️⭐️⭐️⭐️⭐️ | 브라우저와 서버 간에 데이터를 주고받기 위한 HTTP 통신 라이브러리입니다. 기본 `fetch`보다 사용이 편리하고 요청을 가로채거나(Interceptor) 토큰을 제어하기 매우 좋습니다. |

## 4. 💅 CSS 및 UI 스타일링 (Styling)

| 패키지명                             |   중요도   | 설명                                                                                                                                                              |
| ------------------------------------ | :--------: | ----------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **@vanilla-extract/css**             | ⭐️⭐️⭐️⭐️⭐️ | **새로 도입된 핵심 CSS 도구**. 타입스크립트로 CSS를 작성하지만 빌드 시 완벽한 순수 CSS 파일로 변환되어 속도가 아주 빠릅니다(Zero-runtime).                        |
| **styled-components**                |   ⭐️⭐️⭐️   | 기존에 사용되었던 CSS-in-JS 도구입니다. Vanilla Extract로 넘어가는 중이므로 호환성을 위해 당분간 놔두었으나 훗날 완전히 걷어낼 수 있습니다.                       |
| **sanitize.css**                     |   ⭐️⭐️⭐️   | 브라우저(크롬, 사파리, 엣지)마다 기본으로 갖고 있는 CSS 스타일(여백, 폰트 크기 등)이 다른데, 이 찌꺼기들을 "바닐라 상태"로 초기화해서 똑같이 보이게 만들어줍니다. |
| **react-icons**                      |    ⭐️⭐️    | 돋보기, 장바구니, 화살표 등의 아이콘을 이미지(PNG)로 넣지 않고 폰트처럼 코드로 쉽게 가져다 쓸 수 있게 해줍니다. FontAwesome 등을 모아둔 집합소입니다.             |
| **@radix-ui/react-dialog**           |    ⭐️⭐️    | 사용성을 극대화한 모달/다이얼로그 팝업 창을 뼈대(틀)만 제공해 주는 라이브러리입니다. 시각장애인용 스크린 리더기에서도 완벽하게 동작합니다.                        |
| **react-slick** / **slick-carousel** |    ⭐️⭐️    | 메인 페이지 배너 등에서 화면이 옆으로 스르륵 넘어가는 "슬라이더(캐러셀)"를 쉽게 만들게 해주는 라이브러리입니다.                                                   |
| **framer-motion**                    |    ⭐️⭐️    | 메뉴가 부드럽게 나타나거나, 카드가 통통 튀는 등의 '고급 애니메이션'을 아주 쉽게 줄 수 있게 해주는 도구입니다.                                                     |

## 5. 📝 폼 입력 및 검증 (Form & Validation)

| 패키지명                |  중요도  | 설명                                                                                                                                                   |
| ----------------------- | :------: | ------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **react-hook-form**     | ⭐️⭐️⭐️⭐️ | 회원가입, 로그인 폼을 만들 때 입력값 관리, 유효성 검사(에러 메세지), 제출 등을 성능 저하 없이(불필요한 리렌더링 방지) 극도로 쉽게 만들어주는 툴입니다. |
| **zod**                 | ⭐️⭐️⭐️⭐️ | "이메일은 @가 꼭 들어가야 해", "비밀번호는 8자리 이상이어야 해" 같은 **검증 규칙(스키마)**을 타입스크립트와 연동해서 선언하게 해주는 라이브러리입니다. |
| **@hookform/resolvers** |  ⭐️⭐️⭐️  | 위에 설명한 `react-hook-form`과 `zod` 이 둘을 서로 연결해 주는(접착제) 중간 다리 패키지입니다.                                                         |

## 7. 🧪 테스트 (Testing)

| 패키지명                      | 중요도 | 설명                                                                                                                            |
| ----------------------------- | :----: | ------------------------------------------------------------------------------------------------------------------------------- |
| **vitest** / **jsdom**        | ⭐️⭐️⭐️ | 내가 짠 코드가 정상 작동하는지 자동화된 기계가 대신 실행해보는 '테스트 러너'입니다. Jest의 최신/고속 버전입니다.                |
| **@testing-library/react**    | ⭐️⭐️⭐️ | 실제 사용자가 버튼을 클릭하는 것처럼, 화면(UI) 요소들이 제대로 렌더링되고 작동하는지 테스트할 수 있게 해줍니다.                 |
| **msw** (Mock Service Worker) | ⭐️⭐️⭐️ | 진짜 백엔드 서버가 아직 완성되지 않았을 때, **가짜(Mock) API 서버**를 내 브라우저에 임시로 띄워주는 엄청나게 유용한 도구입니다. |

## 8. 🧹 코드 품질 및 포맷팅 (DevTools)

| 패키지명     |  중요도  | 설명                                                                                                                                          |
| ------------ | :------: | --------------------------------------------------------------------------------------------------------------------------------------------- |
| **eslint**   | ⭐️⭐️⭐️⭐️ | 자바스크립트/타입스크립트 코드에서 "안 쓰는 변수 지워", "이 문법 쓰지마"라며 코드의 에러와 나쁜 습관을 짚어내고 강제하는 빨간펜 선생님입니다. |
| **prettier** | ⭐️⭐️⭐️⭐️ | 개발자마다 들여쓰기나 쌍따옴표/홑따옴표 취향이 다릅니다. 이 도구가 저장할 때마다 코드를 **팀 표준에 맞게 깔끔하고 예쁘게 자동 정렬**해줍니다. |
